Manual operativo Menter Fiscal

Usá esta guía durante demos, piloto y operación diaria. La versión Markdown está en docs/MANUAL_USUARIO.md.

Dashboard

Índice

1. Objetivo

Menter Fiscal valida proveedores antes del alta, pago o revisión periódica. Consulta CUITs, cruza fuentes fiscales, genera una decisión sugerida y conserva evidencia en un legajo auditable.

La decisión final siempre corresponde al equipo fiscal/compliance.

2. Roles recomendados

RolUso principal
ComprasConsulta rápida y revisión de decisión sugerida.
Cuentas a PagarValidación masiva y descarga de Excel.
ImpuestosPadrones, fuentes, accesos y observados.
AuditoríaLegajos, evidencia y trazabilidad.
Admin técnicoDiagnóstico, despliegue y monitoreo.

3. Flujo recomendado

  1. Entrar a Dashboard.
  2. Completar Config..
  3. Cargar Padrones críticos.
  4. Registrar Accesos.
  5. Validar desde Consultar o Carga masiva.
  6. Revisar decisión y abrir Legajo.
  7. Atender Cola asistida.

4. Dashboard

Ruta: /dashboard

  1. Abrir Dashboard.
  2. Elegir rol.
  3. Revisar KPIs y riesgo.
  4. Ejecutar próximas acciones.
  5. Abrir últimos legajos si hace falta evidencia.

Interpretación: Aprobables pueden continuar; Observados requieren revisión; Bloqueados no deben aprobarse sin análisis.

5. Configuración guiada

Ruta: /configuracion

  1. Completar cliente/grupo.
  2. Completar CUITs agente.
  3. Marcar jurisdicciones prioritarias.
  4. Guardar configuración local.
  5. Revisar score de cobertura.
  6. Completar checklist: ARCA, padrones, accesos y fuentes.

Score: 85%+ listo para piloto; 60-84% operable con pendientes; menos de 60% requiere configuración.

6. Consulta rápida

Ruta: /

  1. Pegar uno o varios CUITs.
  2. Tocar Validar.
  3. Revisar decisión recomendada.
  4. Abrir detalles si hace falta.
  5. Descargar Excel o abrir legajo.
DecisiónAcción
AprobableContinuar proceso interno.
ObservadoRevisar documentación/fuentes.
Requiere revisiónDerivar a Impuestos.
BloquearNo aprobar sin análisis formal.

7. Carga masiva

Ruta: /lotes

  1. Subir Excel/CSV.
  2. Indicar columna u hoja sólo si no se detectan.
  3. Validar lote.
  4. Descargar Excel o abrir legajo.

Buenas prácticas: columna CUIT clara, archivo limpio y prueba con lote chico si es una fuente nueva.

8. Padrones

Ruta: /padrones

  1. Arrastrar archivo oficial.
  2. Revisar autodetección de provincia/período/vigencia/fuente.
  3. Previsualizar.
  4. Revisar válidos, descartes, duplicados y advertencias.
  5. Importar sólo si la calidad es correcta.

No subir padrones reales a Git. Para archivos grandes usar carga asistida/background o infraestructura mayor.

9. Fuentes

Ruta: /fuentes

  1. Revisar alertas.
  2. Priorizar P0/P1 vencidas o sin carga.
  3. Relevar o descargar cuando sea público.
  4. Enviar a Cola asistida si requiere CAPTCHA, navegador o credenciales.

10. Accesos fiscales

Ruta: /accesos

  1. Revisar pendientes prioritarios.
  2. Usar en formulario para precompletar.
  3. Completar organismo, servicio, tipo, estado, responsable y alcance.
  4. Adjuntar evidencia.
  5. Guardar.

No cargar claves personales en notas. Registrar autorización, exportación o modalidad de acceso.

11. Cola asistida

Ruta: /fuentes-pendientes

  1. Filtrar por estado.
  2. Revisar edad/SLA.
  3. Abrir fuente o legajo.
  4. Cambiar estado.
  5. Agregar nota/evidencia.
  6. Guardar avance.

12. Legajos

Rutas: /legajos y /legajos/{id}

  1. Buscar por CUIT o razón social.
  2. Filtrar por decisión.
  3. Abrir detalle.
  4. Revisar timeline, motivos, recomendaciones y evidencia.
  5. Usar Imprimir/PDF o Excel para soporte documental.

13. Diagnóstico técnico

Ruta: /info

Usar para soporte: modo ARCA, carpetas, padrones detectados, Supabase y alertas básicas.

14. Rutina diaria

Inicio del día

  1. Dashboard.
  2. Bloqueados/observados.
  3. Alertas de fuentes.
  4. Cola asistida.

Antes de aprobar

  1. Consultar CUIT.
  2. Revisar decisión.
  3. Abrir legajo.
  4. Descargar soporte si aplica.

Cierre mensual

  1. Actualizar padrones.
  2. Resolver fuentes vencidas.
  3. Validar lote crítico.

15. Errores frecuentes

SituaciónSolución
CUIT inválidoCorregir CUIT.
Fuente no cargadaIr a Padrones/Fuentes.
Requiere credencialesRegistrar acceso autorizado.
Requiere CAPTCHAResolver desde Cola asistida.
Archivo grande fallaUsar carga asistida/background.
No hay ARCA liveRevisar Diagnóstico técnico.

16. Buenas prácticas

  • Mantener padrones vigentes.
  • Documentar accesos con responsable y alcance.
  • No guardar claves personales.
  • Descargar legajo/Excel para soportes relevantes.
  • Usar cola asistida para tareas manuales.
  • No interpretar “sin hallazgo” como definitivo si una fuente está vencida o faltante.

17. Límites del prototipo

  • Decisión sugerida, no vinculante.
  • Algunas fuentes pueden estar en demo/fallback.
  • PDF es impresión del navegador.
  • Config. guiada usa almacenamiento local.
  • Producción requiere usuarios, roles, auditoría y secretos seguros.
